Vigenere

Vigenere can be considered an improved version of the Caesar code. The main difference is that for the Vigenere code, the shift isn’t the same during the whole text, meaning that the same letter in the original text could be 2 different one after the encryption.
Vigenere, unlike Caesar with a number, uses letters as the encryption key. The key can be a word, a sentence (as long as you don’t use space or punctuation) or a suite of letters without sense. In the key, like in Caesar, each letter is assigned to a number, then the number of each letter is added to the ones in the message you want to encrypt. So, the number of the first letter of the text is added to the number of the first letter of the key, and so on…

Let’s have an example, shall we? Let’s take “The Code” as the message we want to encrypt and “KEY” as the encryption key. Here, we can note that the message is longer than the encryption key. In this case, we just need to repeat the key until it is as long as the message. Our key therefore become “KEYKEYK”.
So now, we just have to add the key’s numbers with the message’s numbers. “K” is the 11th letter of the alphabet and “T” the 20th, so we have 20+10=30. Here, the number is above 26, so we subtract 26 to it : 30-26=4. The 4th letter of the alphabet is “D”, so the first letter of our coded message will be “D”. In the end, your coded message is Dlc Msbo. We can note that the Vigenere code is case-sensitive"case-sensitive" means that the method make a difference between uppercase letters and lowercase letters (unlike morse for exemple). and doesn’t count spaces in the encryption process.
To decrypt a Vigenere code, you almost ABSOLUTELY have to know the key, because the numbers of possible combinations are as big as the number of different possible encryption key, so infinite. Also, as we said before, Vigenere encrypts in a way that a letter in the original message can be many different letters after the encryption, so we can’t find a recurring pattern.
But, if you have the key, then decrypting the message is quite simple. You just have to have the same logic as we said before. You take the numbers of each letter of the encrypted message, and you subtract to it the number of the corresponding encryption key’s letter.
