Sha256

The SHA-256 (Secure Hash Algorithm), on the contrary to encryption, is one of the hash cryptographic function designed by the NSA (National Security Agency), in the USA, and published in 2002 by the government of USA, as FIPS (Federal Information Processing Standard) at the NIST (National Institute of Standards and Technology).
A Hash function is a function which will calculate a unique print (or a signature) from the data provided. Hashing is a function which will "mix" a password, for example, to obtain a print. Using this, the computer will not know your password but it will compare the signature saved on the server with the password you enter, and if it matches, will allow the access.
Properties of hash functions :

1. The length of the signature must always be the same (regardless of the length of the input data. We will see later what are the lengths of the imprints following the hash functions.)
2. Original data cannot be found from the signature: the hash functions only works in one way.
3. It should not be possible to predict a signature. (It is not possible to try to imagine what the signature might be by looking at the data)
4. And finally, of course, for different data: signatures must be different.
A hash function could compare passwords, check downloads data, check the electronic or numeric signature of a file... Also, using a hash function for stocking a big amount of personal data is good because, if the print of the file is present, the document is present. It's faster to check it rather than re-download the whole file.
With the hash function, we could obtain a similar print for a different file. That is called a collision. Generally, that's not a problem because the main goal of a hash function is to compare data and check if the data sent is the same as the data received.
It's a bigger problem if it happens with a hash table or with data stocking. So it's better to use a hash function with a low rate of collisions.

Finally, it's nearly impossible to decrypt a hash function because his role is just to compare data, and you can't recover the original file with the hashed code.
